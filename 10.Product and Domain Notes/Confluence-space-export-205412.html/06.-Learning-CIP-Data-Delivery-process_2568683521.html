Our Git process

eg. All development methodologies (apart from Power BI) leverage off of Bitbucket for version control and release management, as well as Source Tree for development itself.

# Step 1 Analysis: Determine if changes are required.
# If No, Close the tickets make comments
#If yes, proceed to create a feature branch.

# Step 2: Create Feature Branch
git checkout -b feature-branch-name

# Step 3: Develop, Test, and Commit Changes
git add .
git commit -m "Description of changes"

# Step 4: Push Feature Branch and Create Pull Request
git push origin feature-branch-name

# Step 5: Address Peer Review Feedback
# (Make necessary changes, then commit and push)
git add .
git commit -m "Addressed feedback: Description of changes"
git push origin feature-branch-name

# Step 6: Merge to Master
git checkout master
git merge feature-branch-name
git push origin master

Our General workflow

  1. Define types of work:

    1. Dashboard or Report Development >> (what, where is source)

    2. Ad-hoc Data Pull >>(Usually run some sort of sp)

  2. BI Enhancement (New field or Calculation required in DW and BI, it could column evolution too, like name changes, column type changes)

BI Enhancements include:

  • Adding new source tables to the framework

  • Adding new columns to the framework

  • Making amendments to the existing tables or columns within the framework

  1. Automated Data Delivery Process

    1. (SSIS or using Synapse pipeline or even notebooks)

    2. SQL Agent Jobs

    3. Task Schedulers

  2. Stakeholders

    1. Should include reps from each area of the business

    2.  key Technical person (Infrastructure, Old employee that had knowledge)

  3. Prioritization formula for work pipeline (get weightings)

    1. Utility to the business (Ask Why?)

    2. Strategic vs Tactical

      1. Strategic Work:

        • Long-term impact on business goals.

        • Aligns with the company's vision and strategic plan.

        • Typically involves projects that create sustainable competitive advantages.

        • Examples: Implementing a new BI tool, designing a comprehensive data governance framework.

        Tactical Work:

        • Short-term focus, aimed at achieving immediate results.

        • Often operational in nature and supports day-to-day business functions.

        • Examples: Generating weekly sales reports, fixing a bug in a dashboard.

    3. Difficulty/Complexity (how complext)

    4. Estimated Time to Build/implement (How long it going to take)

    5. Regulatory requirement

Important Points

  • Git Workflow: Use Bitbucket for version control and SourceTree for development; follow steps from analysis to merging in Git process.

  • Types of Work: Dashboard/Report Development, Ad-hoc Data Pull, BI Enhancements, Automated Data Delivery Process.

  • BI Enhancements: Include adding new tables/columns, and modifying existing ones within the framework.

  • Automated Processes: Use SSIS, Synapse pipelines, notebooks, SQL Agent Jobs, or task schedulers.

  • Stakeholders: Include business area representatives and key technical persons.

  • Prioritization Formula: Utility to business, strategic vs tactical, difficulty, time to implement, regulatory requirement.

  • Data Sources: SQL Servers (internal/external) and CIP Tabular Models.

  • Data Destinations: Power BI Service, Insights Reports Portal, applications (soon ALOHA), PBI desktop, Excel.

  • Standards for Delivery: Use stored procedures for SQL Server data, Power Query for Tabular Model imports with live connections and custom RLS models.

  • Automated System: Minimize manual checking by using automated processes.

  • RLS Model: Custom RLS roles set by concatenating multiple user attributes, synced daily.

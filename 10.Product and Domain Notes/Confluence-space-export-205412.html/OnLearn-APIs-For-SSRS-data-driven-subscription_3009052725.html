Jackson Fan : OnLearn APIs For SSRS data driven subscription

Created by Jackson Fan, last modified on 10-02-25

Summary:

Dynamic (per recipient) subscriptions are now available for Power BI reports and paginated reports—but only in workspaces that are backed by capacity (such as Microsoft Fabric or Power BI Premium). In a Fabric content workspace, you can set up these dynamic subscriptions to automatically send personalized report copies (filtered for each recipient) via email.

However, this feature isn’t available on the standard Power BI Pro license. To use dynamic subscriptions at the lowest possible cost, you might consider starting with a Microsoft Fabric trial or selecting the entry‑level capacity option that meets your needs.

Below is a high‐level summary along with the key steps to achieve the requirement using our Microsoft Fabric context workspace:

Summary for Your Line Manager:

We want to automatically send two email reminders to any employee who hasn’t completed their mandatory CIP eLearn modules.

The first reminder should be sent one month after the employee’s join date and the second two months after—each email going to the employee and CC’ing their manager.

Using our existing Power BI eLearn Module Completion workspace (which is hosted in Microsoft Fabric/Premium capacity), we can implement this automation by leveraging our Power BI dataset along with a scheduled automation process (such as a Power Automate flow or a Fabric pipeline). This solution will dynamically query our CIP eLearn data to identify employees with outstanding modules and then send personalized reminder emails according to the defined timeframes.

Key Steps to Implement:

  1. Data Source & Model Setup:

    • Confirm Data Availability: Ensure the CIP eLearn dataset in the Power BI workspace contains the following:

      • Employee join date (from the Executive Summary page)

      • Employee email address and manager email address

      • Completion status for each mandatory module (for all employees and, if applicable, additional modules for specific occupations)

      • Occupation (to conditionally check for additional Private Wealth modules)

    • Data Preparation: If needed, create calculated columns or measures in the dataset to flag whether an employee has completed all required modules.

  2. Define Reminder Conditions:

    • Reminder 1: Create a flag (or calculation) that identifies employees whose join date is at least one month ago and who have incomplete eLearn modules.

    • Reminder 2: Similarly, create a flag for employees with a join date of two months (or more) ago who still haven’t completed all the modules.

    • These calculations will drive the selection of recipients.

  3. Automation Setup Using Microsoft Fabric / Power Automate:

    • Scheduled Query: Set up a scheduled process (for example, a daily run using Power Automate or a Fabric pipeline) that queries the dataset to extract the list of employees meeting the Reminder 1 or Reminder 2 conditions.

    • Dynamic Email Trigger: For each employee record that meets the criteria, the process will:

      • Generate an email using the appropriate template (Reminder 1 or Reminder 2).

      • Populate the email subject and body dynamically (for example, “CIP eLearn - outstanding eLearn module(s) to complete” along with the relevant reminder message).

      • Send the email to the employee with the manager’s email added in CC.

  4. Email Template Details:

    • Reminder 1 (1 month post join date):

      • Subject: CIP eLearn - outstanding eLearn module(s) to complete

      • Body: A friendly reminder stating that mandatory modules must be completed within the first month.

    • Reminder 2 (2 months post join date):

      • Subject: CIP eLearn - outstanding eLearn module(s) to complete

      • Body: A second reminder highlighting the urgency if modules are still outstanding.

    • The templates provided will be used as the content of the emails.

  5. Testing and Monitoring:

    • Pilot Test: Run the automation on a test subset of data to verify that the correct employees are identified and the right email is sent at the appropriate time.

    • Deploy & Monitor: Once validated, schedule the process (e.g., daily) and monitor the execution logs to ensure reminders are sent as expected.

Cost Considerations:

  • Capacity Requirement: This solution leverages a Fabric content workspace (or Premium capacity), which is required for dynamic subscriptions or advanced scheduling features.

  • Low-Cost Options: We can start with a Microsoft Fabric trial or choose the lowest‑tier capacity option available to keep costs minimal while meeting the requirements.

This approach will ensure that our Learning and Development team automatically reminds employees (and their managers) about outstanding compulsory eLearn modules based on each employee’s join date, without manual intervention.

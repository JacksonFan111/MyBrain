Jackson Fan : NO/SC Offshore Verification Process from Katrina Gibbs

Created by Jackson Fan, last modified on 29-11-24

Overview

The NO/SC Offshore Verification Report serves as an initial checkpoint for identifying new accounts that require setup in the Offshore RVP/DVP Report.

  • Purpose: Ensure all new offshore accounts are correctly flagged and added to the relevant reports for processing.

  • Frequency: This process typically occurs every 1–2 months when new accounts are created.

  • Key Terms:

    • NO (Nominee): Indicates nominee accounts, marked as 'Y' or 'N'.

    • SC (Safe Custody): Indicates safe custody accounts, marked as 'Y' or 'N'.

    • DVP (Delivery Versus Payment): Ensures securities are delivered only when payment is made.

    • RVP (Receive Versus Payment): Ensures securities are received upon payment confirmation.

Step-by-Step Guide

1. Purpose of the Report

  • The NO/SC Offshore Verification Report identifies new accounts flagged as Nominee (NO) or Safe Custody (SC) that need to be added to the Offshore RVP/DVP Report.

  • Once a new account is processed in the Offshore RVP/DVP Report, it should be excluded from the NO/SC Offshore Verification Report to prevent duplicate notifications.

2. Initial Trigger

  • A user (e.g., Katrina) receives an account from the NO/SC Offshore Verification Report (typically every 1–2 months).

  • The user submits a request to the Data Team to:

    1. Notify that the new account needs to be added to the Offshore RVP/DVP Report.

    2. Request removal (not really remove only adding the Ledger into the Where NOT IN clasue to filter it out)of the account from the NO/SC Offshore Verification Report after it has been added to the Offshore RVP/DVP Report.

  • adding the Ledger into the Where NOT IN clasue to filter it out the account ensures that the verification report does not repeatedly notify users about the same account.

3. Account Addition Process

Step 1: Check Request Details

  • Verify the account number and ledger provided in the request.

  • Example: Contract 12359877 - A/C 276202.

Step 2: Verify the Business Requirement

  • Confirm with the requester (e.g., Katrina) why the account needs to be removed from the report.

  • Ensure the account has been added to the Offshore RVP/DVP Report by the designated person (e.g., Sriyantha).

Step 3: Understand the Report Flow

  • The NO/SC Offshore Verification Report acts as a temporary holding place for new accounts.

  • Once an account is processed in the Offshore RVP/DVP Report, it must be excluded (adding the Ledger into the Where NOT IN clasue) from the Verification Report using a NOT IN clause in the SQL script.

4. SQL Script Adjustments

Step 1: Locate the Current Script

  • Extract the script for the NO/SC Offshore Verification Report from the SSRS server.

  • Example SQL logic:

    SELECT 
    tr.Nominee AS 'Nominee',
    tr.SafeCustody AS 'SafeCustody',
    tr.AccountNumber,
    tr.Ledger
FROM ace.dbo.EqSettleTrans tr
WHERE tr.Ledger NOT IN (<list of processed accounts>)

Step 2: Add the Account to the NOT IN Clause

  • Add the specified Ledger (e.g., 276202) to the NOT IN clause.

  • Updated SQL:

    WHERE tr.Ledger NOT IN ('276202', <other accounts>)

Step 3: Test the Changes

  • Run the updated SQL script and confirm the account no longer appears in the report.

Step 4: Deploy the Changes

  • Save the updated script and deploy it back to SSRS.

  • Notify the user that the account has been successfully removed.

5. Business Context for Each Report

NO/SC Offshore Verification Report

  • Purpose: Acts as a checkpoint to identify new offshore accounts flagged as Nominee or Safe Custody.

  • Outcome: New accounts are flagged and processed in the next report.

Offshore RVP/DVP Report

  • Purpose: Tracks delivery and payment transactions (DVP/RVP) for offshore accounts to ensure accurate settlement and risk management.

6. Communication with Users

  • Maintain proper documentation of why an account was removed for future reference.

  • Example Communication:

    Hi Katrina,  

Your request to remove Contract 12359877 - A/C 276202 from the NO/SC Offshore Verification Report has been actioned.  
This report serves as a checkpoint for adding accounts to the Offshore RVP/DVP Report. Once added, the accounts are excluded from the Verification Report.  

Let us know if you have further queries.  

Cheers,  
[Your Name]

7. Recurring Process

  • Repeat this process whenever a new ledger appears in the NO/SC Offshore Verification Report.

  • Frequency: Typically every 1–2 months.

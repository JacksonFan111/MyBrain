  • Two request type >> Subscription / Interactive

n SQL Server Reporting Services (SSRS), there are two main types of requests: Subscription and Interactive. Here’s a brief explanation of each:

  • Interactive (Request Type 0): This type of request is when a user actively interacts with the report server to request a report. It’s a manual request where the report is rendered in real-time as the user waits1.

  • Subscription (Request Type 1): Subscriptions in SSRS allow reports to be delivered automatically at scheduled times or in response to events. There are two kinds of subscriptions:

    • Standard Subscription: This is a basic subscription where the report is delivered with a fixed set of parameters and options.

    • Data-Driven Subscription: This is a more dynamic subscription that can distribute a report to a fluctuating list of subscribers.

  1. Set Up Your Environment:

    • Ensure you have a Bitbucket account with access to the Craigs.Insights.ReportPortal repository.

    • Install Git Desktop or SourceTree, Visual Studio (with SSRS plugin), and optionally Visual Studio Code.

    • Install Microsoft Report Builder.

  2. Fetch and Pull Changes:

    • Open SourceTree and make sure you’re on the master branch.

    • Fetch the latest changes from the origin to see updates on the remote.

    • Pull the changes to merge them into your local master branch.

  3. Create a New Branch:

    • Use the “Branch” button in SourceTree to create a new branch.

    • Name the branch appropriately, usually including the Jira or Service Desk Ticket reference.

  4. Make Changes to the Report:

    • Modify the report as needed using Visual Studio or Report Builder.

    • Save the changes.

  5. Commit and Push Changes:

    • In SourceTree, stage your changes and commit them with a descriptive message.

    • Push the changes immediately to the origin.

  6. Create a Pull Request:

    • Verify that the new branch is visible in Bitbucket.

    • Create a Pull Request and add a reviewer from the Data Team.

This process ensures that all changes are tracked and reviewed, maintaining the integrity and quality of the reports in Insights.

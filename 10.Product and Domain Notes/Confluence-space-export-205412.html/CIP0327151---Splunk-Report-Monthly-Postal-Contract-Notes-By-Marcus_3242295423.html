Summary of the SOP and Process

  • DGS and SDL Workflow:

    • DGS generates email and postal contract notes. Postal notes are sent to SDL via SFTP (\\svsftp2k8001\soldynamics) as .xml and .pdf files with a control file.

    • SDL processes these files, moves them to a "done" folder, and provides a return control file via email to the CIP Service Desk and SFTP (C:\sftp_root\Usr\soldynamics\SDL return Control Reports).

    • The return control file includes sections: Input Files, Output Channels, and data needing review.

    • Contract notes are processed via:

      • Postal: postaldistributionqueue (SFTP to SDL).

      • Email: emaildistributionqueue (Email Exchange).

    • Failed contract notes (after 5 retries) go to a dead letter queue.

  • Service Desk Responsibilities:

    • Monitor Splunk for "Contract Notes on Dead Letter Queue" > 0:

      • Raise P2 tickets: one for Infrastructure to investigate and release notes from dead queues, one for BA to assist and liaise with the business.

    • Raise P3 tickets for:

      • SQLCIP email ("Unsent Contract Notes – [record(s)]") with count > 0.

      • Report Manager email ("Contract notes sent yesterday") with "Declined" status contract notes.

    • Investigate and resolve issues, involving BA if needed.

  • Infrastructure Responsibilities:

    • Investigate and resolve issues with services (e.g., XpertDoc, Email Exchange, SFTP, SharePoint).

    • Release contract notes from dead letter queues post-resolution.

  • BA Responsibilities:

    • Support Service Desk and Infrastructure in resolving issues.

    • Investigate non-zero counts in SQLCIP’s "Unsent Contract Notes" email using the SQL query:

      SELECT * FROM CdsContractSent WHERE Created >= GETDATE()-1 AND Status <> 'ACCEPTED'

      • Statuses: SENT, DECLINED.

      • DECLINED: If Media = P (postal), Chelmer resends in the next batch. If Media = E (email), reprint from ACE.

      • SENT: Verify if sent (check SharePoint, SFTP, or Mail Marshall). If not sent and Media = E, reprint from ACE. If sent, note details for status update to ACCEPTED.

  • Daily Reports:

    • SQLCIP Report ("Contract notes sent yesterday"):

      • Checks if SENT or DECLINED contract notes are updated to ACCEPTED.

      • Sent daily to Service Desk and nominated recipients.

    • SDL Control File:

      • Sent daily via email/SFTP to CIP.

      • Manually checked for errors by Executive Services (Chontelle).

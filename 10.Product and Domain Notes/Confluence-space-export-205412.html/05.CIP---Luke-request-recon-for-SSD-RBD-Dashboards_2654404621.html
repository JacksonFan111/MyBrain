So when we joining on the [CRM_MSCRM].[dbo].StringMap

FROM [CRM_MSCRM].[dbo].[dsl_portfolioservice] ps

INNER JOIN [CRM_MSCRM].[dbo].StringMap sm
ON sm.AttributeValue = ps.dsl_portfolioservicestatus

--Attribute Value is the columns that we mainly join on
AND sm.AttributeName = 'dsl_portfolioservicestatus

--AttributeName is the column that we wan to filter to the specific table
AND sm.ObjectTypeCode=10042

-- although not required but good to always use ObjectTypeCode check while using StringMap table

Be aware of these parameters:

  • sm.AttributeValue >> --Attribute Value is the columns that we mainly join on with the dsl_table >> eg. ON sm.AttributeValue = ps.dsl_portfolioservicestatus

  • sm.AttributeName, >> this is the Column name that we want to join with >> eg. AND sm.AttributeName = 'dsl_portfolioservicestatus'
    Why do we filter on the column name >> it is becasue that string maps is a huge table with a lot of different columns name value in it

  • sm.ObjectTypeCode,>> this ObjectTypeCode is to filter to make sure the records obejcts is current
    eg. AND sm.ObjectTypeCode=10042 -- although not required but good to always use ObjectTypeCode check while using StringMap table

  • sm.Value, >> this is the actual desctive value we would like to retrive as a text value >> this is the human undertandable apart

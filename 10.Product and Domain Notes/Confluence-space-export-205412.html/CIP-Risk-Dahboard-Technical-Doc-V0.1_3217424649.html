0 Background

Overview

Jackson Fan (Development) and Andy Cheang (Compliance) successfully led the renewal of the CIP Risk Dashboard, with the final version launched on Power BI Services on June 24, 2025. After 29 design iterations, the dashboard effectively visualizes the Compliance team’s Key Risk Indicator (KRI) metrics.

Key Features

  • Metrics Structure: Displays Risk Area, Metric, Metric Data Type, Metric Value, KRI Score, and KRI Description.

  • Scoring Mechanism: Assigns scores to various data types based on manually defined lower and upper bounds.

  • Risk Descriptions: Provides clear, concise text summaries for each risk.

Implementation Details

Due to CIP data access policies, the solution relies on manually curated “Smartsheets” as the data source. Occasional user-driven changes, such as sheet relocations or column renaming, reflect the dynamic environment. Despite these challenges, the dashboard delivers a practical and effective tool for the Compliance team.

Current Limitations and Future Opportunities

At this stage, transforming the descriptive bounds in “Smartsheets” into a qualitative database or a comprehensive data model was not feasible. However, the current solution lays a solid foundation for future improvements, and the team is committed to ongoing support and optimization.

Conclusion

The renewed CIP Risk Dashboard represents a significant achievement, balancing complex data constraints with user needs. We’re excited about its impact and look forward to further enhancing its capabilities.

1 Purpose & Core Goal

The CIP Risk Dashboard unifies Key Risk Indicator (KRI) data from Smartsheet, an on‑prem SQL Server 2008 database, and two existing Power BI semantic models. The goal is to provide a continuously refreshed, drill‑able view of organisational risk across 15 domains, replacing nine Smartsheets packs to have one central place to look at KRI.

5.2 Sample M – KRI Register

let
    Source = Smartsheet.Tables(),
    Sheet  = Source{[Id="7870718639044740"]}[Data],
    Promote= Table.PromoteHeaders(Sheet, [IgnoreErrors=true]),
    Types  = Table.TransformColumnTypes(Promote,{
              {"KRI No", type text},
              {"Score", Int64.Type},
              {"Updated", type datetime}}),
    Loaded = Table.AddColumn(Types, "[Data Loaded]", each DateTime.LocalNow(), type datetime)

    Loaded

6 DAX Measures & Calc Groups

6.1 Naming Convention

[BusinessContext] [Metric] [Agg] → e.g., Cyber Incident Count YTD, Supplier Spend 12M Rolling.

6.2 Core Measures (selection)

Risk Score Avg = AVERAGE( 'Key Risk Indicator Score'[Score] )

Risk Score Normalised = DIVIDE( [Risk Score Avg], 10 )

Current Risk Rating =
SWITCH( TRUE(),
    [Risk Score Avg] < MIN( 'Risk Threshold'[Low] ), "Low",
    [Risk Score Avg] < MIN( 'Risk Threshold'[Med] ), "Medium",
    "High" )

High Priority Action Count =
CALCULATE( COUNTROWS('ActionRegister'), 'ActionRegister'[Status] = "High" )

Incident Count 12M =
CALCULATE( [Incident Count], DATESINPERIOD('CalendarDate'[Date], MAX('CalendarDate'[Date]), -12, MONTH) )

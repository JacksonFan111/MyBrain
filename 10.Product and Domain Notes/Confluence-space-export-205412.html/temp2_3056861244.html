I01: QuayStreet Asset Names in CIP and MT Don’t Match

Objective: Identify clients with QuayStreet assets where names differ between CIP and MT environments, using the provided query and QuayStreet fund list.

/*
Purpose: Identify clients impacted by mismatched QuayStreet asset names between CIP and MT.
Assumption: ProductName in Asset table reflects CIP naming; MT naming aligns with 'QuayStreet [Fund] Units PIE' from the document.
*/

SELECT DISTINCT
    'I01 - QuayStreet Asset Name Mismatch' AS Issue,
    e.Id AS EntityId,
    e.EntityName AS ClientName,
    p.Id AS PortfolioId,
    p.PortfolioReference AS PortfolioCode,
    a.PieFundID,
    a.ProductName AS CIP_ProductName,
    CASE 
        WHEN a.ProductName NOT LIKE '%Units PIE' THEN CONCAT(a.ProductName, ' Units PIE')
        ELSE a.ProductName
    END AS Expected_MT_ProductName,
    CASE 
        WHEN a.ProductName NOT LIKE '%Units PIE' THEN 'Mismatch Detected'
        ELSE 'Match'
    END AS MatchStatus
FROM 
    Asset a
    LEFT JOIN Portfolio p ON p.Id IN (
        SELECT Portfolio_Id FROM PortfolioAsset WHERE Asset_Id = a.Id
    )
    LEFT JOIN Entity e ON e.Id = p.Client_Id
WHERE 
    a.ProductName LIKE '%Quaystreet%'
    AND a.PieFundID IN (330001, 330002, 330003, 330004, 330005, 330006, 330007, 330008, 330009, 330010, 330011, 330012) -- From QuayStreet fund list
    AND a.ProductName NOT LIKE '%Units PIE' -- Assume MT names end with "Units PIE" per document
ORDER BY 
    a.PieFundID, e.Id;

Notes:

  • Enhanced the original query (select a.PieFundID, a.status, a.RecordStatus, a.SourcesystemGuid, a.* from Asset a where ProductName like '%Quaystreet%') with joins to Portfolio and Entity.

  • Assumes MT names should match the format in the fund list (e.g., "QuayStreet Conservative Fund Units PIE").

  • Outputs a list of clients with mismatched asset names for manual correction.

I03: Incorrect CountryCity in CIP (Sydney as NZ City)

Objective: Identify clients with Sydney listed as a New Zealand city in CIP, using the provided CountryCity query.

/*
Purpose: Identify clients with Sydney incorrectly mapped to New Zealand in CIP ContactDetail.
Assumption: ContactDetail.City and CountryCity should align; Country_Id 2071 = NZ, 1922 = Australia.
*/

SELECT DISTINCT
    'I03 - Incorrect CountryCity (Sydney as NZ)' AS Issue,
    e.Id AS EntityId,
    e.EntityName AS ClientName,
    p.Id AS PortfolioId,
    p.PortfolioReference AS PortfolioCode,
    cd.City AS ContactDetailCity,
    cd.Country_Id AS ContactDetailCountryId,
    c.CountryName AS ContactDetailCountry,
    cc.Id AS CountryCityId,
    cc.Country_Id AS CountryCityCountryId,
    cc_c.CountryName AS CountryCityCountry
FROM 
    ContactDetail cd
    LEFT JOIN Entity e ON e.PostalAddress_Id = cd.Id OR e.PhysicalAddress_Id = cd.Id
    LEFT JOIN Portfolio p ON p.Client_Id = e.Id
    LEFT JOIN Country c ON c.Id = cd.Country_Id
    LEFT JOIN CountryCity cc ON cc.CityName = cd.City
    LEFT JOIN Country cc_c ON cc_c.Id = cc.Country_Id
WHERE 
    cd.City = 'Sydney'
    AND cc.Id = 1673 -- Sydney mapped to NZ per document
    AND cd.Country_Id = 2071 -- NZ country ID
ORDER BY 
    e.Id;

Notes:

  • Builds on select * from CountryCity where CityName='Sydney' and select * from Country where id in (1922,2071).

  • Links ContactDetail to Entity and Portfolio to identify affected clients.

  • Filters for Sydney (ID 1673) incorrectly tied to NZ (2071), not Australia (1922).

I06: Historical Fee Transactions Linked to MyStart Fee Package No Longer in Use

Objective: Identify clients with historical fee transactions tied to an obsolete MyStart fee package.

/*
Purpose: Identify clients with historical fees linked to an obsolete MyStart package.
Assumption: RecordStatus = 2 indicates "no longer in use"; MyStart identified by context.
*/

SELECT DISTINCT
    'I06 - Obsolete MyStart Fee Transactions' AS Issue,
    e.Id AS EntityId,
    e.EntityName AS ClientName,
    p.Id AS PortfolioId,
    p.PortfolioReference AS PortfolioCode,
    pmf.Id AS ManualFeeId,
    fied.Id AS FeeDetailId,
    pmf.Description AS FeeDescription
FROM 
    PortfolioManualFee pmf
    INNER JOIN FeeIncomeAndExpenseDetail fied ON pmf.Id = fied.PortfolioManualFeeItem_Id
    INNER JOIN Portfolio p ON p.Id = fied.Portfolio_Id
    INNER JOIN Entity e ON e.Id = p.Client_Id
WHERE 
    fied.Portfolio_Id IN (
        SELECT Id FROM ReplicationSelectedPortfolio
    )
    AND pmf.RecordStatus = 2 -- No longer in use per workaround precedent
ORDER BY 
    p.Id, e.Id;

Notes:

  • Uses the original query (select a.* from PortfolioManualFee a join FeeIncomeAndExpenseDetail b on a.Id = b.PortfolioManualFeeItem_Id where b.Portfolio_Id in (select Id from ReplicationSelectedPortfolio)).

  • Assumes MyStart is identified by context; filters for RecordStatus = 2 (inactive).

  • Links to Entity for client details.

I08: Duplicate BranchNumber in MT

Objective: Identify clients impacted by duplicate BranchNumber (0936) in MT.

/*
Purpose: Identify clients linked to duplicate BranchNumber '0936' in MT.
Assumption: Current workaround sets RecordStatus = 2 for ID 28262; we identify active duplicates.
*/

SELECT DISTINCT
    'I08 - Duplicate BranchNumber' AS Issue,
    e.Id AS EntityId,
    e.EntityName AS ClientName,
    p.Id AS PortfolioId,
    p.PortfolioReference AS PortfolioCode,
    bb.BranchNumber,
    bb.BranchName,
    bb.Id AS BankBranchId,
    bb.RecordStatus
FROM 
    BankBranch bb
    LEFT JOIN Portfolio p ON p.OrganisationBranch_Id = bb.BranchID -- Assuming OrganisationBranch_Id links to BranchID
    LEFT JOIN Entity e ON e.Id = p.Client_Id
WHERE 
    bb.BranchNumber = '0936'
    AND bb.RecordStatus = 1 -- Active records only, excluding workaround (ID 28262)
    AND bb.Id != 28262 -- Exclude the workaround ID
ORDER BY 
    e.Id;

Notes:

  • Enhances Select RecordStatus, * from BankBranch where BranchNumber = '0936'.

  • Assumes duplicate active records (RecordStatus = 1) need identification; excludes ID 28262 per workaround.

  • Links to Portfolio and Entity for client impact.

I09: Entity Primary Contact with Non-Craigs Organisation ID

Objective: Identify entities with primary contacts tied to non-Craigs Organisation IDs.

/*
Purpose: Identify entities where primary contact has a non-Craigs Organisation_Id.
Assumption: Craigs Organisation_Id = 1; others are non-Craigs.
*/

SELECT DISTINCT
    'I09 - Non-Craigs Organisation ID' AS Issue,
    e.Id AS EntityId,
    e.EntityName AS ClientName,
    e.Organisation_Id AS EntityOrganisationId,
    CASE 
        WHEN e.Organisation_Id != 1 THEN 'Non-Craigs'
        ELSE 'Craigs'
    END AS OrganisationStatus
FROM 
    Entity e
WHERE 
    e.Id IN (49189, 45162, 45553, 37838, 43051, 31682) -- From document
    AND e.Organisation_Id != 1 -- Non-Craigs
ORDER BY 
    e.Id;

Notes:

  • Uses select Organisation_Id, * from Entity where id in (49189,45162,45553,37838,43051,31682) as a base.

  • Assumes Craigs Organisation_Id = 1 (common default); filters for non-Craigs IDs.

  • Simple output for manual correction.

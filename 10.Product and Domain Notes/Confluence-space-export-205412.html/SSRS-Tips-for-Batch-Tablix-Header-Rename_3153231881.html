Why this matters
When SSRS auto-sanitises special characters ( / ' . ) in column headers you can end up with cryptic titles such as Substantial U S Owner s Type. Fixing them directly in the report designer is painful and brittle; editing the report RDL (XML) is faster and fully repeatable.

  1. Export the RDL

    Open your report ‚ûú Download ‚ûú Save the .rdl file.

    • Optional: copy the XML into a .docx if you prefer Copilot‚Äôs Word plug-in.

  2. Keep Dataset Fields Untouched üîí 

    <Field Name="Substantial_US_Owner_s_Type">
    <DataField>Substantial_US_Owner_s_Type</DataField>
</Field>

    Do not remove the underscores; SSRS binds on these field names.

  3. Fix Headers in the <TablixBody> 

    <TablixCell>
    <CellContents>
        <Textbox Name="Substantial_US_Owner_s_Type">
            <TextRuns>
                <TextRun>
                    <!-- Before -->
                    <Value>Substantial U S Owner s Type</Value>
                    <!-- After  -->
                    <Value>Substantial U.S. Owner's Type</Value>
                </TextRun>
            </TextRuns>
        </Textbox>
    </CellContents>
</TablixCell>

    Pro-tip:
    Use a regex replacement:

    Find :  ([A-Z])\sU\sS\s
Replace: $1 U.S.

  4. Audit Column Count

    Quick check ‚Üí grep -c "<TablixCell" should return 127.
    If not, undo the last edit (you probably deleted a closing tag).

  5. Re-deploy & Test

    Upload the RDL ‚ûú Run the report ‚ûú Confirm the headers display with U.S. / ' / . characters intact.

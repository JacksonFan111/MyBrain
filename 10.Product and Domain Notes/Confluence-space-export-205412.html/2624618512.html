Key Concepts

  1. Effective Rate: It considers the impact of compounding over time. The effective rate is often higher than the nominal rate due to the effects of compounding. >> Effective is a concept that bring in the time dimension

  2. Daily Average Portfolio Value: average value of the portfolio over a given period, which helps normalize other financial metrics for comparison.

Measures Explained

1. Effective Fee Rate >>> AKA. Average Fee Rate in Luke Context

Definition:

MEASURE 'Asset Transaction'[Effective Fee Rate] =
    CALCULATE ( [Fees Value Gross of GST] / [Daily Average Portfolio Value] )

Explanation:

  • Fees Value Gross of GST: This is the total fees collected, including Goods and Services Tax.

  • Daily Average Portfolio Value: This is used to normalize the fees value by the average value of the portfolio.

Example:

Gross means the total, whole or complete

Net means what’s left after deductions
If the total fees collected (gross of GST) over a month is $10,000 and the daily average portfolio value is $1,000,000, the effective fee rate would be:
10,000 / 1,000,000 = 1%

2. Effective Brokerage Rate

Definition:

MEASURE 'Asset Transaction'[Effective Brokerage Rate] =
    CALCULATE ( [Transaction Brokerage] / [Daily Average Portfolio Value] )

Explanation:

  • Transaction Brokerage: The total brokerage fees from transactions.

  • Daily Average Portfolio Value: This normalizes the brokerage fees by the average portfolio value.

Example:
If the total brokerage fees over a month are $5,000 and the daily average portfolio value is $1,000,000, the effective brokerage rate would be:
=$5,000/$1,000,000=0.005=0.5%

3. Effective Revenue Rate

Definition:

MEASURE 'Asset Transaction'[Effective Revenue Rate] =
    CALCULATE ( [Revenue Gross] / [Daily Average Portfolio Value] )

  • Revenue Gross: The total revenue generated before .

4. Daily Average Portfolio Value

Definition:

MEASURE 'Asset Transaction'[Daily Average Portfolio Value] =
    CALCULATE (
        SUM ( 'TWR by Portfolio'[ClosingAmountNZD] ) / COUNT ( 'TWR Date'[TWR Date] ),
        LASTNONBLANK (
            'TWR Date'[TWR Date],
            CALCULATE ( SUM ( 'TWR by Portfolio'[ClosingAmountNZD] ) )
        )
    )

Explanation:

  • SUM ( 'TWR by Portfolio'[ClosingAmountNZD] ): Sums up the closing amounts for the portfolio in NZD.

  • COUNT ( 'TWR Date'[TWR Date] ): Counts the number of dates to find the average.

  • LASTNONBLANK: Ensures that only non-blank values are considered for the calculation.

Example:
If the closing amounts for the portfolio over 30 days are $30,000,000 and there are 30 days in the period, the daily average portfolio value would be:
Daily Average Portfolio Value} = ($30,000,000/30 Days)=$1,000,000

Use cases:

  • Effective Fee Rate shows how much the company is earning in fees relative to the size of the portfolio.

  • Effective Brokerage Rate indicates the cost incurred from brokerage fees in relation to the portfolio.

  • Effective Revenue Rate provides insight into the overall profitability relative to the portfolio size.

  1. Revenue Net: This column from Asset Transaction fact table >>The net revenue generated by the adviser from managing client funds after deducting the Cost.
    Certainly! Let's break down the measures and format them nicely with explanations and examples.

Base Measures

  1. Fees Net Of GST

MEASURE 'Asset Transaction'[Fees Net Of GST] = 
    CALCULATE(
        [Fees Value Gross of GST] - [Fees GST Amount]
    )

  • Explanation: This measure calculates the net fees by subtracting the Goods and Services Tax (GST) amount from the gross fees value.

  • Example:

    • If [Fees Value Gross of GST] is $100 and [Fees GST Amount] is $10, then [Fees Net Of GST] would be $90.

  1. Brokerage Net Agency Fees

MEASURE 'Asset Transaction'[Brokerage Net Agency Fees] = 
    CALCULATE(
        [Transaction Brokerage] - [Agency Fees]
    )

  • Explanation: This measure calculates the net brokerage by subtracting agency fees from the transaction brokerage.

  • Example:

    • If [Transaction Brokerage] is $50 and [Agency Fees] is $5, then [Brokerage Net Agency Fees] would be $45.

Combined Measure

  1. Revenue Net

MEASURE 'Asset Transaction'[Revenue Net] =
    CALCULATE(
        [Fees Net Of GST] + [Brokerage Net Agency Fees],
        PATHCONTAINS("S|T", 'Asset Transaction Status'[Transaction Status Code])
    )

  • Explanation: This measure calculates the net revenue by summing up the net fees (excluding GST) and the net brokerage (excluding agency fees). The calculation is filtered to include only transactions with a status code containing "S" or "T".

  • Example:

    • Suppose [Fees Net Of GST] is $90 and [Brokerage Net Agency Fees] is $45.

    • The PATHCONTAINS function checks if the transaction status code contains "S" or "T".

    • If the status code condition is met, then [Revenue Net] would be $90 + $45 = $135.

Rolling Average Gross Amount NZD

This measure calculates the rolling average of the Gross Amount NZD up to the current order.

Rolling Average Gross Amount NZD = 
VAR CurrentRow = MAX('Orders'[Order Number])
VAR RowsToConsider = 
    FILTER(
        'Orders',
        'Orders'[Order Number] <= CurrentRow 
    )
RETURN
    AVERAGEX(
        RowsToConsider, 
        'Orders'[GrossAmountNZD]
    )

Important Concepts from CRM:

A. there is a table called Portfolio Service Level

The level of service (or "product") given to a client's Portfolio Service to manage it's fee levels, among other settings.

Service Levels while not strictly matching CIP's range of products 1-for-1 are as close enough to think of as what type of service we as a company will apply to the assets inside

This is an important entity and contains links to outside services, eg. Chelmer and the AACPR databases

dsl_portfolioservicelevel

B. one level up is called Portfolio Service

Portfolio Service (dsl_portfolioservice)

this table contain most of the data.

Explanation: A client's financial portfolio, containing holding information and a product ("service level") that governs how the assets are managed and charged for

Multiple Portfolio Services can exist under a single Portfolio

Holdings are assigned to a Portfolio Service as well as a Portfolio, this is because while a parent Portfolio can hold many assets / holdings they can be separated into different Portfolio Services for the purpose of having different management schemes (or products, called service levels) apply to individual assets

A Portfolio Service will usually have a Portfolio Service ID, which is unique, this ID is sent up to the owning Trading Entity as a value in Portfolio ID's Searching to assist finding accounts by Portfolio Service ID

File Notes and Contract Notes link here because these type of communications usually refer to a specific set of holdings that the Portfolio Service will be managing.

C. An entity is a table of data within the CRM database

When a user is looking at a record in the CRM UI it is data retrieved behind-the-scenes from the entity that stores it. These are essentially tables, and can be accessed in much the same way as any SQL-based data structure eg. by query to a view or the table directly

Each entity has a unique number assigned to it, referenced by CRM as an Object Type Code

Custom entities are prefixed with some letters, whereas out-of-the-box ones are not

For example, the entity activitypointer is for Activities and is part of standard CRM, whereas cip_webroleallocation has “cip” at the front to mean that a staff member of CIP added it

The prefixes in use are:

cip = Craigs

dsl = Datacom

cdi = Click Dimensions

xt = Xpertdocs

Each entity has a dedicated Confluence page, view the A-Z list on the left-hand-side of this page or search for the one you are looking for

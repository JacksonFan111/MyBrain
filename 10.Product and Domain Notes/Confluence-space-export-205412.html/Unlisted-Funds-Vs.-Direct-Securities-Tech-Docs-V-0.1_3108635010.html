Business Background

Core Goal

Advisor Client Classification Measures (Unlisted Funds Vs. Direct Securities) Enable CIP to evaluate each advisor’s client mix by quantifying what proportion of their billed clients hold—and pay for—active direct securities versus unlisted funds versus a combination of both. This helps ensure advisors are providing the investment management service their clients are paying for.

Context

  • Advisors charge higher fees for active portfolio management (Portfolio Service).

  • Some Advisors’s clients may channel most assets into passive, unlisted funds to minimize hands-on work.

  • We need to spot whether an advisor’s book is overly tilted toward unlisted funds (i.e. low-touch) rather than active direct security selection.

Challenges

  1. Overlapping Holdings: Many clients hold both direct securities and unlisted funds, so simple filters can misclassify them.

  2. Data Grain: The fact table is at the Account × Asset + FUM level; classification must roll up correctly to each advisor’s book.

  3. POWER BI DirectQuery Mode: No calculated columns; all logic must be written as measures that translate efficiently back to the SQL source. (In the Enhancement in SSAS we could just created the columns then just count the columns do the Rows Count)

Classification Definitions & Examples

Before we jump into DAX, let’s define exactly what we mean by each category—and illustrate with real-world scenarios.

1. Direct Securities Only

  • Definition: Clients whose portfolios contain only individual stocks, bonds, ETFs or other exchange-listed instruments. No managed or private funds.

  • Example Assets:

    • NZX-listed shares (e.g. FPH, AIA)

    • Government or corporate bonds

    • Exchange-traded funds (ETFs)

  • Use Case:

    • An advisor builds a bespoke portfolio of 10 New Zealand shares and 5 corporate bonds for a client who wants hands-on stock selection and pays active management fees.

2. Unlisted Funds Only (QuayStreet, SmartShares Etc.)

  • Definition: Clients whose portfolios contain only private or non-exchange-traded funds—often higher-fee, unlisted managed funds (NZUL. NZUT and AUUT).

  • Example Assets:

    • NZUT and NZUL unlisted property or private equity funds

    • AUUT Australian unlisted trusts

  • Use Case:

    • A retirement-focused client places all savings into an unlisted KiwiSaver fund (market code NZUT) for simplicity. The advisor’s role is primarily administrative, not stock selection.

3. Mixed Clients

  • Definition: Clients whose portfolios include both direct securities and unlisted funds.

  • Example Assets:

    • A mix of NZX shares (direct) plus a 20% allocation to an NZUT unlisted property fund

    • Combination of ASX shares and an AUUT private equity trust

  • Use Case:

    • A growth-oriented investor asks the advisor to pick individual stocks for 80% of assets, and “park” the remaining 20% in a low-volatility unlisted fund.

    • This reflects a hybrid service: active stock picking plus passive fund administration.

V1 Design: Classification at the Account Level

1. Base Measure: Current FUM Value

MEASURE 'Funds Under Management'[Current FUM Value] =
CALCULATE(
    SUM([FUM Value]),                                    // 1) Sum all holdings
    'Account Holdings Date'[last holding date] = "yes"    // 2) Only the latest date
)

2. Category Counts

2.1 Direct Securities Only

So in V1 -- I used a 10 Measures Approaches to represent the the % of advisers per portfolio Services level

We are count of the [AccountKey] For V1 because I think that was used to Linked with [Fund under management] table

I have checked the Portfolio Service , Portfolio Service ID, And then The Account Key is at the same Data Level of the Grain which represent the Clients in CIP context.

# Count Direct Securities Clients =
CALCULATE(
    DISTINCTCOUNT('Account'[AccountKey]),                 // A) count each client once
    FILTER(
        'Funds Under Management',                        // B) examine each holding row
        NOT ISBLANK([Current FUM Value])                 // 1) has non-blank assets
        && NOT(RELATED('Asset'[Market Code])             // 2) exclude unlisted-fund markets
            IN {"NZUT","NZUL","AUUT"})
        && NOT(RELATED('Asset'[Asset Type])              // 3) exclude Fund & Cash Asset Type
            IN {"Cash","Fund"})
    )
)

2.2 Unlisted Funds Only

# Count Unlisted Funds Clients =
CALCULATE(
    DISTINCTCOUNT('Account'[AccountKey]),
    FILTER(
        'Funds Under Management',
        NOT ISBLANK([Current FUM Value])                    // 1) has assets
        && RELATED('Asset'[Asset Type]) = "Fund"            // 2) is a Fund
        && RELATED('Asset'[Market Code])                    // 3) in unlisted markets
            IN {"NZUT","NZUL","AUUT"}
    )
)

2.3 Mixed Clients

# Count Mixed Clients =
SUMX(
    VALUES('Account'[AccountKey]),                        // 1) for each client...
    VAR acct = 'Account'[AccountKey]                      //    store key
    VAR hasDir =                                          // 2) check direct holdings
        COUNTROWS(
            FILTER(
                'Funds Under Management',
                [AccountKey] = acct
                && NOT ISBLANK([Current FUM Value])
                && RELATED('Asset'[Asset Type]) <> "Fund"
                && RELATED('Asset'[Asset Type]) <> "Cash"
            )
        ) > 0
    VAR hasFun =                                          // 3) check unlisted funds
        COUNTROWS(
            FILTER(
                'Funds Under Management',
                [AccountKey] = acct
                && NOT ISBLANK([Current FUM Value])
                && RELATED('Asset'[Asset Type]) = "Fund"
                && RELATED('Asset'[Market Code])        // in unlisted markets
                   IN {"NZUT","NZUL","AUUT"}
            )
        ) > 0
    RETURN
      IF(hasDir && hasFun, 1, 0)                         // 4) 1 if both true
)

2.4 Total Valid Clients

# Valid Client Accounts =
[# Count Direct Securities Clients]
+ [# Count Unlisted Funds Clients]
+ [# Count Mixed Clients]

3. Percentage Measures

# % Mixed Clients = DIVIDE([# Count Mixed Clients], [# Valid Client Accounts], 0)
# % Direct Securities Only Clients = DIVIDE([# Count Direct Securities Clients], [# Valid Client Accounts], 0)
# % Unlisted Funds Only Clients = DIVIDE([# Count Unlisted Funds Clients], [# Valid Client Accounts], 0)

V2 Design: Flags at the Portfolio Service ID Level

1. Base Asset Flags

#Count Direct AssetID =
CALCULATE(
    DISTINCTCOUNT('Funds Under Management'[AssetID]),
    'Asset'[Asset Type] <> "Fund"
    && 'Asset'[Asset Type] <> "Cash"
)

#Count Fund AssetID =
CALCULATE(
    DISTINCTCOUNT('Funds Under Management'[AssetID]),
    FILTER(
        ALL('Asset'),
        Asset[Asset Type] = "Fund"
        && Asset[Market Code] IN {"NZUT","NZUL","AUUT"}
        && CONTAINSSTRING(Asset[Asset Name], "FUND")
        && NOT ISBLANK([Current FUM Value])
    )
)

#ISDirect = IF([#Count Direct AssetID] > 0, 1, 0)
#ISFund   = IF([#Count Fund AssetID]   > 0, 1, 0)
#ISMixed  = IF(AND([#ISDirect] = 1, [#ISFund] = 1), 1, 0)

2. Final Counts by Flags

# MixedV2 =
COUNTROWS(
  FILTER(
    ADDCOLUMNS(
      VALUES('Account'[Portfolio Service ID]),
      "IsDirect", CALCULATE([#ISDirect]),
      "IsFund",   CALCULATE([#ISFund]),
      "IsMixed",  CALCULATE([#ISMixed])
    ),
    [IsMixed] = 1
  )
)

# DirectOnlyV2 = COUNTROWS(FILTER(..., [IsDirect] = 1 && [IsMixed] = 0))
# FundOnlyV2   = COUNTROWS(FILTER(..., [IsFund]   = 1 && [IsMixed] = 0))

# ValidClientsV2 = [MixedV2] + [DirectOnlyV2] + [FundOnlyV2]

DirectQuery Considerations

  1. No calculated columns – all logic must be measures.

  2. Context transitions – CALCULATE + FILTER push filters to the source; SUMX/ADDCOLUMNS iterate row by row.

  3. Performance – simpler filters translate more efficiently to SQL.

Conclusion

These annotated measures show how to classify advisor portfolios under DirectQuery. Students should now understand:

  • How to filter and count at different granularities.

  • The difference between filter context and row context.

  • How to write efficient DirectQuery-friendly DAX.

  • # Valid Client Accounts:
    The overall universe of accounts with non‐blank FUM.

  • # Count Unlisted Funds Clients:
    All valid accounts that have at least one Unlisted row (Equity + Fund + Market Code in {NZUT, NZUL}).

  • # Count Direct Securities Clients:
    All valid accounts that have at least one Direct row (i.e., do not match the Unlisted criteria).

  • Overlap (Mixed):
    Accounts that appear in both sets—i.e., hold at least one Unlisted row and at least one Direct row.

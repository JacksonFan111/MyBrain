Our Naming Convention

Stored Procedures:

Naming conventions:

  • dbo.usp_

  • Schema can change depending on whether or not it is logically grouped with other similar procs of a single business function.

eg.

dbo.usp_GetCustomerDetails

hr.usp_UpdateEmployeeSalary

sales.usp_InsertNewOrder

SSIS Packages:

Storage locations:

  • Can be stored on the file server or on the Server’s C drive.

  • Best practice would be on the SQL file server.

eg. SSIS package for exporting customer data

File Server: \\FileServer\SSIS\CustomerDataExport.dtsx

C Drive: C:\SSIS\CustomerDataExport.dtsx

SQL Agent Jobs:

Naming conventions:

  • [Process Grouping] - [Specific Process Name]

  • i.e. Compliance - Wilsons Positions Import

For jobs relating to loggin/monitoring, please use:

  • Logging - [Specific Process being logged]

  • i.e Logging - Server Health Check

Important Points

  1. Overview: CIP's integration processes are outdated, requiring an understanding of various legacy and new technologies.

  2. Technologies: Utilizes SQL Agent Jobs, SSIS Packages, Task Scheduled Jobs, and PowerShell.

  3. History: Older integrations are on SVSQLCIP\SQLCIP and SVSCHED2k8001, with Karl DBA having full access.

  4. New Integrations: Hosted on KAPSQLDAT-XXX.

  5. Naming Conventions: Stored procedures use dbo.usp_, SSIS packages should be stored on the file server, and SQL Agent Jobs follow a specific naming format.

  6. SSIS Catalogue: Contains a comprehensive list of processes, both old and new, managed via SQL Agent Jobs.

  7. Documentation: Chris's documentation and project links provide detailed functional specifications for key processes.

  8. Resource Limitation: There is a shortage of resources to fully address underperforming client portfolios.

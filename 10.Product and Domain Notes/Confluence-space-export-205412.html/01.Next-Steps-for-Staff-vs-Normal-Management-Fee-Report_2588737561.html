Background

The goal is to compare the management fees charged to staff members versus the fees that would be charged if they were not staff.

essentially it is comparing percentage of Management fee apply to Staff and Percentage of fee that would be charged if they were not staff

This comparison requires understanding the financial impact of staff discounts on management fees. The report will use selection criteria such as start date, end date, and portfolio ID, displaying columns including client code, client name, staff type, branch, adviser code, staff management fee, and normal management fee.

Challenges

  1. Staff Management Fee: Pre-calculated and stored in the PortfolioCashTrade table as pct.AmountBaseCCY AS 'Staff Management Fee'.

  2. Normal Management Fee: Needs to be calculated based on specific fee packages which are not fully provided, except for the superSTART Fee Schedule.

Next Steps

  1. Data Extraction:

    • Develop SQL queries to extract the NZD valuation for each portfolio and break them down into cash, shares, and units.

    • Example SQL for extracting valuation data:

    SELECT 
    p.PortfolioID,
    SUM(CASE WHEN a.AssetType = 'Cash' THEN a.ValuationNZD ELSE 0 END) AS CashValuation,
    SUM(CASE WHEN a.AssetType = 'Shares' THEN a.ValuationNZD ELSE 0 END) AS SharesValuation,
    SUM(CASE WHEN a.AssetType = 'Units' THEN a.ValuationNZD ELSE 0 END) AS UnitsValuation
FROM 
    Portfolios p
    JOIN Assets a ON p.PortfolioID = a.PortfolioID
GROUP BY 
    p.PortfolioID;

  2. Fee Schedule Data:

    • Work with the vendor (NZXWT) to add actual fee schedule data into the dbo.PortfolioFeePackage table.

    • Create a request for this data as a structured table.

  3. Calculation of Normal Management Fee:

    • Using the fee schedules, calculate the daily fee assessment for each portfolio based on its valuation in different asset types.

    • Sum these daily fees over the specified period to get the total normal management fee.
